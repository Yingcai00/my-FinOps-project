# 定价管理 · 成本关联校验（只读报告 + 跳转）

## 1. 文档概述

### 1.1 文档目的

本文档定义在 **定价管理页面**（`pricing-management.html`）内增加的 **成本关联校验** 能力的设计方案与需求规格，用于：

- 在 **不引入新数据模型、不新增持久化结构** 的前提下，基于现有 **成本池** 与 **服务规格** 数据做只读分析；
- 通过 **统计看板 + 问题列表 + 一键跳转筛选/打开编辑** ，帮助用户发现「成本池未挂到规格/规格簇」「引用不一致」等问题，降低定价时成本遗漏风险。

### 1.2 适用范围

- 前端：单页 `pricing-management.html` 内的 UI 与脚本扩展（可与现有 `localStorage` 成本池、`serviceData` 同源读取逻辑对齐）。
- **Demo 页**：`pricing-management-cost-linkage-demo.html` 在定价页基础上实现「成本关联检查」抽屉及「账期出账就绪统计」等交互，便于评审与验收对齐。
- 不包含：后端接口、数据库表结构、金额级分摊算法、批量写回新字段。

### 1.3 参考文档

- 项目内《企业私有云FinOps系统 计量管理页面需求文档》（文档结构参考）
- 项目内 `cost-pool-management.html`（成本池数据来源约定）
- 项目内 `pricing-management.html`（服务目录、筛选、编辑弹窗现有行为）

### 1.4 术语定义

| 术语 | 定义 |
|------|------|
| 成本池 | 成本归集单元，名称与 `localStorage` 中 `costPools[].name` 一致；含计费资源、总价、账期等 |
| 服务规格 | 定价表中的一行，对应 `serviceData` 中单条记录，含规格编码、规格名称、规格簇等 |
| 规格簇 | 本页面中与三级分类一致，对应字段 `categoryLevel3` |
| 关联成本池 | 服务规格上填写的成本池名称，对应字段 `costPool`（字符串） |
| 账期 | 页面筛选使用的月份，与 `filterMonth` 及服务数据 `month` 对齐统计 |

## 2. 业务背景与目标

### 2.1 背景

定价依赖成本池单价与资源量，若部分成本池未关联到任何可定价的服务规格（或规格簇下全部未关联），则定价结果可能 **遗漏整块成本** 。用户需要在定价前快速 **自检** ，并能 **连续处理** 多条规格，而非仅靠人工扫表。

### 2.2 本期目标（MVP）

1. **只读报告**：根据当前账期与内存/本地成本池数据，生成统计与明细列表，不写新存储。
2. **可行动**：从报告项可 **跳转** 到已有「筛选结果」或 **打开已有编辑弹窗** ，减少查找路径。
3. **无新数据模型**：不新增 `localStorage` key、不新增服务端实体；仅消费现有 `costPools` 与 `serviceData` 字段。

### 2.3 非目标（本期不做）

- 按用量/资源量将成本池金额 **拆分到多条规格** 并校验加总为 100%。
- 批量保存、审批流、版本快照。
- 独立新 HTML 文件（本期以 **定价页内嵌模块** 为默认落地形态；若产品坚持独立页，可作为二期，本文档规则仍复用）。

## 3. 总体方案设计

### 3.1 方案选型

采用 **「只读报告 + 跳转」**：

| 组成 | 说明 |
|------|------|
| 统计区 | 关键指标卡片或紧凑条，数字来自实时计算（打开报告时或账期变更时刷新） |
| 列表区 | 分 Tab 或分节展示多类问题/提示，每行可操作 |
| 跳转 | 复用页面已有 `searchServices` / 筛选 DOM / `editService(id)`（或等价函数），不复制业务保存逻辑 |

### 3.2 在页面中的信息架构（建议）

在 **账期选择行** 同一视觉区域右侧，或 **服务列表卡片标题行** 右侧，增加：

- 主按钮：**「成本关联检查」**  
- 点击后展开 **抽屉（Drawer）** 或 **全宽折叠面板**，内嵌：统计 + 列表；关闭不改变已筛选状态（除非用户点击了「应用筛选」类操作）。

> 选用抽屉可避免挤压下方表格初始布局；若实现更简单也可用 `modal` 大号弹窗，需求上等价。

### 3.3 数据流（逻辑）

```
filterMonth（页面控件）
    ↓
costPools = localStorage['costPools'] 解析结果（与成本池管理页一致）
serviceData = 当前页内存中的服务数组（与列表同源）
    ↓
校验引擎（纯函数）→ 结构化结果 { summary, issues[] }
    ↓
渲染报告 UI；用户点击「跳转」→ 设置筛选条件并触发 searchServices / 打开编辑弹窗
```

**一致性要求**：统计所用的 `serviceData` 必须与表格渲染同源（若存在从 `localStorage` 加载服务数据的逻辑，则报告与表格应共用同一加载结果）。

| FR-02-02 | 未被任何规格引用的成本池数 | 不存在满足条件的规格：`service.month === filterMonth` 且 `trim(service.costPool) === 池名称`（大小写/空格策略见 5.2） | P0 |
| FR-02-03 | 未填写关联成本池的规格数 | `month === filterMonth` 且 `costPool` 为空或仅空白 | P0 |
| FR-02-04 | 孤儿引用规格数 | `costPool` 非空但该字符串 **不匹配** 当前快照中任一成本池 `name` | P0 |
| FR-02-05 | 计费资源不一致数（可选 MVP+） | `costPool` 能匹配到池，但 `service.billingResource` 与池的 `billingResource` 在规范化后不一致 | P1 |
| FR-02-06 | 规格簇「全空」簇数（可选） | 某 `categoryLevel3` 下，该月至少有一条规格，且 **所有** 该簇规格的 `costPool` 均为空 | P1 |

## 5. 校验规则细则（供开发对齐）

### 5.1 账期对齐

- 主筛选器：`#filterMonth` 的值，格式 `YYYY-MM`。
- 服务侧：仅统计 `service.month === filterMonth` 的记录（字段名以现有 `serviceData` 为准）。
- 成本池侧：优先 `pool.costMonth === filterMonth`；若对象无 `costMonth`，**策略 A（推荐）**：该池仍计入「当期应管池」但在 UI 显示角标「未标月份」；**策略 B**：全部池参与（需在 FR-02 注明）。**本文档推荐策略 A**，避免多月池干扰「当月定价」语义。

### 5.2 字符串匹配

- `costPool` 与成本池 `name` 比较前：`trim()`；是否大小写敏感默认 **敏感**（与中文场景一致）；若历史数据存在大小写混杂，可在实现备注中增加 `toLowerCase` 选项（默认关闭）。

### 5.3 「未覆盖成本池」判定

对每一个参与统计的成本池 `P`：

- 若不存在规格 `s` 满足：`s.month === filterMonth` 且 `trim(s.costPool) === trim(P.name)`，则 `P` 列入 **未覆盖成本池**。

### 5.4 「未关联规格」判定

- `s.month === filterMonth` 且（`!s.costPool` 或 `trim(s.costPool) === ''`）。

### 5.5 「孤儿引用」判定

- `trim(s.costPool) !== ''`，且该字符串不等于任一参与统计的成本池 `name`（在 trim 后比较）。

### 5.6 「计费资源不一致」（可选）

- 存在池 `P` 使得 `trim(s.costPool) === trim(P.name)`，规范化 `s.billingResource` 与 `P.billingResource` 后二者不等。  
- 规范化规则需与页面现有 `updateBillingResource` 或展示层一致（例如全角半角、`MEM`/`内存` 同义是否在 MVP 合并，由实现与产品确认，文档默认 **字面一致**）。

### 5.7 「未计费成本项」判定

- **业务定义**：未关联成本池的 **非公摊** 成本项。  
- **原型/Demo 实现口径**：成本项数据优先读取 `localStorage['costItems']`（与 `cost-detail.html` 等写入约定一致）；若无数据则使用页面内嵌演示数据。  
- **「已关联成本池」判定**：成本项 `id` 出现在 **任一参与当期统计的成本池** 的 `selectedItems` 数组中（与 `cost-pool-management.html` 维护方式一致）。  
- **非公摊**：`shareType === '非公摊成本'` 或文案中包含「非公摊」（可按产品收口为枚举精确匹配）。  
- **账期**：若成本项存在 `costMonth` 字段，则仅统计 `costMonth === filterMonth`；若无 `costMonth`，Demo 中视为 **参与所有账期**（正式环境建议强制补全 `costMonth` 以免跨月误计）。

### 5.8 「未计费成本池」判定

- **业务定义**：没有服务规格关联的成本池。  
- **实现口径**：与本文 **5.3「未覆盖成本池」** 一致——参与统计的成本池 `P` 下，不存在当月服务 `s` 满足 `trim(s.costPool) === trim(P.name)`。

## 6. 交互与视觉（概要）

- **入口**：主色线框按钮或文字按钮 + 图标（可选），文案固定为「成本关联检查」。
- **面板**：宽度建议 480px～640px（抽屉）；或高度不超过视口 85% 的弹窗。
- **统计**：一行 3～4 个指标，未覆盖池、未关联规格、孤儿引用为必显；其余折叠为「更多指标」。
- **列表**：表格密度与主表接近；操作列固定右侧；移动端非本期重点。
- **空状态**：各分节独立空状态文案。

## 8. 验收标准（测试要点）

1. 修改 `localStorage.costPools` 或页面数据后，打开检查面板，**未覆盖成本池**列表与手工核对一致。  
2. 将某规格 `costPool` 置空，**未关联规格**列表包含该条；点击「编辑」弹出与主表一致的编辑弹窗且为同一 `id`。  
3. 将某规格 `costPool` 设为不存在的名称，**孤儿引用**列表包含该条。  
4. 点击「筛选此成本池」后，主表筛选结果仅展示该池（或与现有多筛选组合逻辑一致，需在测试用例中写明预期）。  
5. 切换账期后，各统计数字与列表随 `month` / `costMonth` 规则变化，无 JS 报错。  
6. 账期模块：**未计费成本项数** 与 **未计费成本池数** 与按 5.7、5.8 手工核对一致；任一项大于 0 时出现「无法出账」提示条。  
7. 点击统计卡片展开左侧明细；**弹窗选择关联** 后 `localStorage.costPools` 或 `serviceData` 更新正确，统计归零或相应减少，主表筛选/展示一致。
