## 1. 背景与目标

### 1.1 背景

- 需在**产品层**统一：**角色**、**功能权限**（菜单/按钮/流程）、**数据权限**（组织或资源可见范围），并支持**多租户**隔离叙事。
- 本文档所列菜单为当前 FinOps 产品范围内**纳入权限管控**的模块；未列入的页面不作为本文验收对象。

### 1.2 目标

- 多租户下各租户数据与权限边界清晰，可培训、可验收。
- **功能**与**数据**规则分离表述：先有功能入口，再在范围内裁剪数据。
- **导出、重跑、推送等原「可配置」能力**：一律作为**功能权限项**在矩阵中定义；租户内是否收紧，通过**同一套功能权限配置**（租户管理员对「角色 × 功能项」的覆盖）完成，**不再单独建「租户策略」章节**。

### 1.3 设计原则

- **默认最小权限**。
- **租户内自治**：租户管理员在**不突破本规格各角色默认上限**的前提下，可关闭部分功能项（见第 4 节说明）。
- **数据范围优先于界面**：无入口不等于安全；正式环境以服务端规则为准（验收必须覆盖接口/数据层）。
- **多角色并存**：功能与数据范围均取**并集**；与租户内功能权限覆盖冲突时，**以不允许为准**（禁止通过并集放大租户已关闭的能力）。

### 1.4 与技术实现的关系（摘要）

- **多租户数据隔离**：业务数据需按租户分区（如租户标识及统一访问约束），与授权判定分离。
- **授权判定**：可采用独立策略引擎等实现，本文不绑定具体技术选型；产品验收以**本文矩阵与用例**为准。

## 2. 术语

| 术语 | 说明 |
|------|------|
| 租户 | 客户侧独立 FinOps 空间；组织、用户、账单、配置等均从属于租户。 |
| 组织 / 部门树 | 含多级部门；用户有主岗部门；可有「部门负责人」身份。 |
| 本人资源 | 与当前登录用户存在**资源归属绑定**的云资源或台账行（绑定规则由租户在资源/主数据中配置）；账单与明细仅展示与这些资源相关的部分。 |
| 角色 | 由租户管理员分配给用户；每个角色绑定一组**功能权限项**（见第 4 节矩阵默认值）。 |
| 功能权限 | 对某一**菜单或按钮级能力**是否允许；含导出、重跑、推送等，均在功能权限矩阵中逐条控制。 |
| 租户内功能权限覆盖 | 租户管理员在角色默认矩阵基础上，对本租户**关闭**某些功能项（仅允许从「允许」收紧为「不允许」，不得放宽超过矩阵默认值）。 |
| 数据权限 | 在具备功能权限前提下，列表/汇总/图表/导出**内容**允许出现的组织或资源范围（见数据权限表）。 |

## 3. 角色定义（租户级）

| 角色 | 典型使用者 | 职责摘要 |
|------|------------|----------|
| 租户管理员 | IT / 平台 owner | 租户内用户与角色、组织与安全策略；系统管理；B 类资产、成本、定价、分账等配置类能力；**为本租户配置各角色功能权限的允许/不允许覆盖**（不得突破第 4 节矩阵上限）。 |
| 私有云运营管理员 | 私有云运营 / 财务对账 / 成本治理 | 全租户成本与账单、分账标签维护、定价查看等；具体能执行的重跑、推送等以第 4 节矩阵及租户内覆盖为准。 |
| 部门负责人 | 部门负责人 | **仅**账单详情、账单分析相关功能项；无成本、定价、分账、系统管理等菜单能力。 |
| 业务用户 | 研发 / 业务 | 与部门负责人相同的功能项范围（账单详情、账单分析）；数据范围为**本人资源**（见数据权限表）。 |

**系统级（跨租户）角色**（若存在）：与上述租户级角色严格区分；所有操作强制审计；本文不展开跨租户流程。

**交互说明（建议）**：部门负责人、业务用户登录后，侧栏**仅展示「账单管理」**及其子菜单；验收以服务端鉴权为准。

## 4. 菜单与功能权限（中文总表）

### 4.1 图例（表内用语）

| 用语 | 含义 |
|------|------|
| **无** | 无菜单入口或无该能力。 |
| **允许** | 具备该功能权限（若同时受数据权限约束，内容范围见数据权限表）。 |
| **不允许** | 不具备该功能权限。 |
| **允许（租户可关闭）** | 角色默认**允许**；租户管理员可在「功能权限 / 角色权限」配置中**对本租户关闭**，关闭后该角色在本租户内视为**不允许**（不得对其它租户生效）。 |
| **允许（租户可收紧）** | 角色默认**允许**；租户管理员可在本租户内将执行方式收紧为例如「仅租户管理员可执行」「私有云运营管理员可执行」「需双人确认后方可执行」等**具体实现由功能权限配置项枚举**，收紧后仍属「允许」的一种受控形态；若配置为「关闭」则等同**不允许**。 |
| **查看** | 进入菜单后的只读浏览（列表、详情、筛选、图表只读），不含导出、重跑等行上单独列出的能力。 |
| **维护** | 可新增、修改、删除该菜单下的配置或主数据。 |
| **查看与维护** | 只读与维护类能力均具备。 |

**一致性**：同一用户多角色时，对同一功能项取**并集**；若租户已对某角色关闭该项，该用户在该租户内**不得**因兼任其他角色而重新获得该项（**就严**，见 1.3 节）。

### 4.2 功能权限矩阵（默认值）

下列为**角色默认**；租户管理员仅可按下文「允许（租户可关闭）」「允许（租户可收紧）」及说明进行**关闭或收紧**，不得将「不允许」改为「允许」，不得将「无」改为「允许」。

| 功能项 | 租户管理员 | 私有云运营管理员 | 部门负责人 | 业务用户 |
|--------|-------------|------------------|------------|----------|
| B 类资产管理—查看与维护 | 查看与维护 | 查看 | 无 | 无 |
| 成本管理 / 成本总览—查看与维护 | 查看与维护 | 查看 | 无 | 无 |
| 成本管理 / 成本总览—导出 | 允许 | 允许 | 无 | 无 |
| 成本管理 / 成本池管理—查看与维护 | 查看与维护 | 查看 | 无 | 无 |
| 定价管理—查看与维护 | 查看与维护 | 查看 | 无 | 无 |
| 分账管理—查看与维护 | 查看与维护 | 查看与维护 | 无 | 无 |
| 系统管理 / 用户、角色、配置—查看与维护 | 查看与维护 | 无 | 无 | 无 |
| 账单管理 / 账单详情—进入与查看 | 允许 | 允许 | 允许 | 允许 |
| 账单管理 / 账单详情—导出 | 允许 | 允许 | 允许（租户可关闭） | 允许（租户可关闭） |
| 账单管理 / 账单详情—重跑账单 | 允许（租户可收紧） | 允许（租户可收紧） | 不允许 | 不允许 |
| 账单管理 / 账单详情—推送账单 | 允许（租户可关闭） | 允许（租户可关闭） | 不允许 | 不允许 |
| 账单管理 / 账单分析—进入与查看 | 允许 | 允许 | 允许 | 允许 |
| 账单管理 / 账单分析—导出 | 允许 | 允许 | 允许（租户可关闭） | 允许（租户可关闭） |

**矩阵说明**

- **部门负责人**、**业务用户**：上表中凡标记为「无」的菜单整体不可见；**仅**与「账单详情」「账单分析」相关的行可能为「允许」。
- **成本总览—导出**：仅租户管理员、私有云运营管理员两行；与「成本总览—查看与维护」区分，便于租户单独关闭导出而保留查看（若产品支持拆配）。
- **重跑账单 / 推送账单**：「允许（租户可收紧）」表示租户可在功能权限配置中选择：关闭；或仅租户管理员；或允许私有云运营管理员；或双人确认等枚举，**均在本功能项上配置**，不另立策略文档章节。

### 4.3 租户内功能权限覆盖（配置边界）

- **配置入口**：与「角色权限」同一产品能力（名称以实际 UI 为准），配置维度为：**租户 × 角色 × 功能项**（功能项名称与第 4.2 节各行一致）。
- **允许的操作**：将第 4.2 节中「允许（租户可关闭）」项关闭为不允许；将「允许（租户可收紧）」项配置为具体收紧模式或关闭。
- **禁止的操作**：将默认「不允许」改为「允许」；为部门负责人/业务用户开通成本、分账等非账单功能项。

## 5. 数据权限（独立表）

**范围类型说明**

| 数据范围类型 | 含义 |
|--------------|------|
| 全租户 | 租户内全部组织及对应账单、成本、资源等数据。 |
| 本部门及下级 | 以用户主岗部门为根，包含组织树上该节点及全部下级节点对应的数据。 |
| 本部门不含下级 | 仅当前组织节点对应的数据。 |
| 指定部门集合 | 由租户管理员为某用户单独配置的可访问部门白名单。 |
| 本人资源 | 仅与当前用户已绑定的资源相关的账单、明细与汇总（绑定规则由租户配置）。 |

**角色与默认数据范围**

| 角色 | 默认数据范围 | 补充说明 |
|------|----------------|----------|
| 租户管理员 | 全租户 | 不因部门身份缩小。 |
| 私有云运营管理员 | 全租户 | 成本、账单、分账等数据与导出范围均为全租户（导出仍须第 4.2 节对应功能项为允许）。 |
| 部门负责人 | 本部门及下级 | 仅账单相关功能项生效时适用；列表、图表、下钻、导出内容不得超出该范围。 |
| 业务用户 | **本人资源** | 仅账单相关功能项生效时适用；仅展示与本人绑定资源相关的数据与导出。 |

**一致性（验收）**

- 在「账单详情」与「账单分析」之间：凡带组织或资源归属筛选的，**分析页可见范围不得大于详情页**。

## 6. 例外与流程（产品要求）

| 场景 | 规则 |
|------|------|
| 临时跨部门查看 | 走申请—审批；审批人建议为**租户管理员**或**私有云运营管理员**；不改变部门负责人默认仅账单功能项的产品定位；数据扩大须配套临时授权，且仍受已允许的功能项约束。 |
| 组织调整（合并 / 迁移） | 须在专项需求中约定：历史账单按**当时组织快照**或按**新组织重归属**，二者不可混用表述。 |
| 关账后改标签 / 补录 | 建议状态机：**租户管理员**与**私有云运营管理员**可发起更正类流程（具体以关账产品规则为准）。 |

## 7. 审计与体验

- **建议记录审计日志的操作**：登录租户、**功能权限覆盖变更**、角色分配变更、导出、重跑 / 关账、分账主数据变更、跨部门临时授权生效与失效。
- **无权限提示**：区分「无功能权限」与「无数据范围权限」，便于用户联系管理员。
- **正式环境**：不以页面上的临时角色切换作为权限依据（原型仅供演示）。

## 8. 验收用例（节选）

1. **部门负责人**：侧栏仅账单管理；第 4.2 节中账单查看为允许、导出为「允许（租户可关闭）」时默认可导出；租户关闭导出后按钮不可用且接口拒绝。
2. **部门负责人**：访问成本总览等 URL：**无权限**（对应功能项为无）。
3. **业务用户**：账单仅本人资源；导出与列表一致；重跑、推送为不允许。
4. **私有云运营管理员**：重跑、推送为「允许（租户可收紧）」时，租户配置为关闭后不可执行；配置为双人确认时流程未通过前不可执行。
5. **多角色用户**：功能项并集；租户已对某角色关闭的功能项，兼任后仍**不允许**（就严）。

## 9. 假设与待扩展

- 用户默认只属于一个租户；跨租户访问不在本文展开。
- 「部门负责人」与 HR 主岗部门一致；**本人资源**绑定关系由资源或主数据同步提供，需单独字段说明。
